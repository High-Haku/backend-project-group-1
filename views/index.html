      Introduction

      Haku Library RESTful API, merupakan sebuah API yang menyediakan berbagai macam data buku, mulai dari informasi penulisnya, penerbitnya sampai harga buku.

      Ada beberapa route path yang harus anda ketahui untuk mengakses datanya. Namun anda perlu login terlebih dahulu agar dapat mengakses API kami. Apabila anda telah login, anda akan diberi sebuah Bearer Token yang nantinya digunakan untuk authentikasi dalam mengakses data kami. Berikut user demo yang bisa anda gunakan untuk mengakses API.

      Demo Account

      email : budi@mail.com
      pass : 12345

      Route Path

      Perlu diperhatikan tidak semua route path bisa diakses oleh semua orang. Ada beberapa route path yang hanya bisa diakses oleh role tertentu. User dibedakan menjadi 3 role : admin, user, guest(tidak login).

      https://haku-library-api.herokuapp.com/{{path}}

      Route Path Deskripsi Akses
      /books Mendapatkan Seluruh Data Buku admin (crud)
      user (read)
      /writers Mendatapkan Seluruh Data Penulis admin (crud)
      user (read)
      /publishers Mendapatkan Seluruh Data Penerbit admin (crud)
      user (read)
      /users Melihat Semua Data User Yang Terdaftar admin (crud)
      user (read)
      /transactions Melihat Semua Transaksi admin (crud)
      user (read & update)
      /login Halaman Login Untuk Mendapatkan Token / Redirect ke Halaman Admin guest
      /views Halaman Admin, Untuk Mengelola Data API admin

      GET Datas

      Berikut route path yang bisa diambil datanya dengan method GET :

      /books
      /writers
      /publishers
      /transactions
      /users

      Ada beberapa cara dalam pengambilan data, yaitu mengambil data secara keseluruhan, mengambil data berdasarkan ID. Dan khusus untuk route path books anda dapat mengambil data yang diingankan dengan memanfaatkan url query.

      Search Book By URL Query

      Khusus untuk route path books, anda bisa mencari buku berdasarkan query, adapun penggunaanya tambahkan “?” setelah route path books diikuti key dan value nya. Dan jika lebih dari satu query tambahkan “&” untuk menyambungnya.

      Contoh

      /books?{key}={value}
      /books?genre=comedy
      /books?title=a&maxprice=100000

      Berikut tabel dari query-nya yang bisa anda gunakan.

      Key Deskripsi
      title Mencari Berdasarkan Judul Buku
      genre Mencari Berdasarkan Genre
      price Mencari Berdasarkan Harga Buku
      minprice Mencari Buku Dengan Harga Diatas Minimal
      maxprice Mencari Buku Dengan Harga Dibawah Maksimal
      stock Mencari Berdasarkan Jumlah Stock Buku
      minstock Mencari Buku Dengan Jumlah Stock Diatas Minimal
      maxstock Mencari Buku Dengan Jumlah Stock Dibawah Maksimal
      writer Mencari Buku Berdasarkan Penulis
      publisher Mencari Buku Berdasarkan Penerbit
      releaseDate Mencari Buku Berdasarkan Tanggal Terbit

      Adding & Updating Data

      Yang diperlu diperhatikan dalam menambahkan data baru maupun update data adalah schema dari collection. Request Data susunannya harus sesuai dengan schema collection tujuan. Nama property, tipe data, maupun besar kecil huruf harus sesuai dengan schema nya. Apabila salah maka server akan merespon error 400 bad request.

      Untuk menambahkan data gunakan method POST ke alamat tujuan

      https://haku-library-api.herokuapp.com/{{path}}

      Namun untuk update data gunakan method PUT ke alamat tujuan diikuti dengan ID data yang ingin di update

      https://haku-library-api.herokuapp.com/{{path}}/:id

      Data Schema

      Untuk meminimalisir kesalahan dalam menambahkan data maupun meng-update data berikut schema dari setiap collection.

      Books Schema

      title : string, (required)
      writer : string,
      publisher: string,
      genre : string, (required)
      description : string,
      price : number, (required)
      stock : number, (required)
      releaseDate : date,
      img : image file (jpg,jpeg,png)

      Writers Schema

      name : string, (required)
      dateOfBirth : date,
      location : string,
      books: bookID

      Publishers Schema

      name : string, (required)
      location : string,
      books : bookID,

      Transactions Schema

      products: bookID, (required)
      totalPrice: number,
      status: enum (proses, done, canceled, pending, sending), (required)
      purchaseDate: date, (auto fill)

      Users Schema

      name: string, (required)
      email: email, (required)
      password: string, (required)
      role: enum (user, admin), (required)
      image: image file (jpg,jpeg,png),
      wishlist: bookID,
      purchaseHistory: transactionID,

      Delete Data

      Untuk delete data gunakan method DELETE ke alamat tujuan diikuti dengan ID data yang ingin di delete. Perlu diingat juga user yang bisa melakukan delete hanyalah user dengan role admin.

      *Data yang sudah di delete tidak dapat dikembalikan lagi.
